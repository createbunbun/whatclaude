💻 คล็อดโค้ด

คล็อด โค้ด

ตัวแทน AI ที่ทุ่มเทให้กับการเขียนโค้ด มันทำงานจากเทอร์มินัล (บรรทัดคำสั่ง)

🤖
การเข้ารหัสอัตโนมัติ

Claude จะสร้าง แก้ไข ลบไฟล์ รันคำสั่ง ทดสอบ ฯลฯ โดยอัตโนมัติ เพียงบอกให้ "สร้างแอพนี้" แล้วมันจะเขียนโค้ดโดยอัตโนมัติ

🔍
ความเข้าใจโค้ดเบส

คุณสามารถโหลดโฟลเดอร์โปรเจ็กต์ที่มีอยู่และทำความเข้าใจโค้ดทั้งหมดก่อนที่จะแก้ไขข้อบกพร่อง ปรับโครงสร้างใหม่ หรือเพิ่มคุณสมบัติต่างๆ

⚡
การดำเนินการคำสั่ง

ดำเนินการคำสั่งโดยอัตโนมัติ เช่น การติดตั้ง npm, git และสคริปต์ทดสอบ คุณสามารถทำให้ขั้นตอนการพัฒนาเป็นแบบอัตโนมัติตามที่เป็นอยู่ (ต้องมีการยืนยัน)

🌐
ค้นหาเว็บ/อ้างอิงเอกสาร

สร้างโค้ดในขณะที่ค้นหาข้อมูลห้องสมุดและเอกสารประกอบล่าสุด เป็นไปได้ที่จะดำเนินการตามข้อมูลล่าสุด

🚀 การติดตั้ง/ตั้งค่า

⚠️
วิธีใช้รหัสคลอดด์ คีย์ API ของมานุษยวิทยา(คิดค่าบริการ) จำเป็นคู่มือ API ดูวิธีรับกุญแจ

สภาพแวดล้อมที่จำเป็น

Node.js 18 หรือใหม่กว่า npm หรือ npx macOS / Linux / Windows (แนะนำ WSL) คีย์ API ของมานุษยวิทยา

คำแนะนำในการตั้งค่า

1

ติดตั้ง Node.js

nodejs.org ดาวน์โหลดและติดตั้งเวอร์ชัน LTS จาก

2

ติดตั้งรหัสคลอดด์

$ การติดตั้ง npm -g @anthropic-ai/claude-code เพิ่ม 248 แพ็คเกจใน 12 วินาที
3

ตั้งค่าคีย์ API เป็นตัวแปรสภาพแวดล้อม

#macOS/ลินุกซ์ $ ส่งออก ANTHROPIC_API_KEY='sk-ant-...' #วินโดวส์พาวเวอร์เชลล์ > $env:ANTHROPIC_API_KEY = 'sk-ant-...'
4

เริ่มต้นในโฟลเดอร์โครงการ

$ cd ~/projects/my-app $ คลอดด์ ✓ รหัสคลอด v1.x.x ไดเร็กทอรีการทำงาน: /Users/you/projects/my-app เราจะทำอย่างไร?

💡 ตัวอย่างการใช้งาน

สร้างแอปใหม่

คุณ: สร้างแอปพยากรณ์อากาศด้วย React สิ่งง่ายๆ ที่ใช้ OpenWeather API เพื่อแสดงสภาพอากาศในตำแหน่งปัจจุบันของคุณ โคลด์: เข้าใจแล้ว สร้างแอป React พยากรณ์อากาศ ✓ กำลังตรวจสอบโครงสร้างโครงการ... ✓ สร้าง src/App.jsx... ✓ การสร้าง src/components/WeatherCard.jsx... ✓ สร้าง src/hooks/useWeather.js... ✓ กำลังอัปเดต package.json... ✓ เสร็จแล้ว! คุณสามารถเริ่มต้นด้วย 'npm start'

แก้ไขข้อบกพร่อง

คุณ: `TypeError: ไม่สามารถอ่านคุณสมบัติ 'id' ของไม่ได้กำหนด` ปรากฏขึ้นเมื่อเข้าสู่ระบบ แก้ไขมัน คลอดด์: ตรวจสอบข้อผิดพลาด ✓ กำลังโหลด src/auth/login.js... ✓ พบปัญหา: ไม่มีการตรวจสอบค่าว่างสำหรับวัตถุผู้ใช้ ✓ กำลังแก้ไข src/auth/login.js... ✓ แก้ไขแล้ว! การเปลี่ยนแปลง: - ก่อน: const userId = user.id + หลังจากแก้ไข: const userId = user?.id ?? โมฆะ

การปรับโครงสร้างใหม่

คุณ: ฟังก์ชั่นที่ซ้ำกันใน utils.js จัดระเบียบ. คลอดด์: วิเคราะห์โค้ด ✓ แยกวิเคราะห์ utils.js (384 บรรทัด)... ✓ พบรายการที่ซ้ำกัน: formatDate ถูกกำหนดสองครั้งใน 3 ตำแหน่ง ✓ รวมเข้ากับฟังก์ชันทั่วไป... ✓ การรีแฟคเตอร์เสร็จสมบูรณ์: 384 บรรทัด → 201 บรรทัด (ลดลง 47%)

ทดสอบการสร้าง

คุณ: เขียนการทดสอบ Jest สำหรับฟังก์ชันทั้งหมดใน src/api/userService.js คลอดด์: สร้างแบบทดสอบ ✓ กำลังแยกวิเคราะห์ userService.js... ✓ ตรวจพบ 7 ฟังก์ชั่น ✓ สร้าง tests/api/userService.test.js... ✓ สร้างกรณีทดสอบ 23 กรณี ✓ การตั้งค่าจำลอง... ✓ เสร็จแล้ว! คุณสามารถรันมันได้ด้วย `npm test`
